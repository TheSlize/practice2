# Задание 1.2 — Таблица модификаторов доступа

## Теоретическое напоминание

В Java существуют четыре уровня доступа:

| Модификатор | Свой класс | Тот же пакет | Наследник (другой пакет) | Весь мир |
|-------------|:----------:|:------------:|:------------------------:|:--------:|
| `private`   |     ✅     |      ❌      |           ❌             |    ❌    |
| *(без модификатора)* |  ✅  |   ✅   |           ❌             |    ❌    |
| `protected` |     ✅     |      ✅      |           ✅             |    ❌    |
| `public`    |     ✅     |      ✅      |           ✅             |    ✅    |

**Важно для этого задания:** класс `Employee` находится в пакете `company.core`, а класс `HRSystem` — в пакете `company.app`. Это **разные** пакеты. `HRSystem` **не** наследует `Employee`.

## Ваша таблица

Для каждой строки (A–H) из `HRSystem.java` укажите:

| Строка | Обращаемый член | Модификатор | Компилируется? (да/нет) | Причина |
|--------|----------------|-------------|:-----------------------:|---------|
| A | `emp.name` | `public` | **да** | `public` члены доступны из любого места (любого пакета и любого класса). |
| B | `emp.age` | `protected` | **нет** | `protected` доступен только в том же пакете или классам-наследникам. `HRSystem` находится в другом пакете и не наследует `Employee`. |
| C | `emp.salary` | *(без модификатора) / package-private* | **нет** | Доступ по умолчанию (package-private) позволяет обращаться к полю только из того же пакета (`company.core`). `HRSystem` находится в другом пакете (`company.app`). |
| D | `emp.password` | `private` | **нет** | `private` члены доступны исключительно внутри самого класса, в котором они объявлены (`Employee`). |
| E | `emp.getRole()` | `public` | **да** | `public` методы доступны из любого пакета и класса без ограничений. |
| F | `emp.promote(5000)` | `protected` | **нет** | Аналогично строке B: метод `protected` недоступен, так как вызывающий класс находится в другом пакете и не является наследником. |
| G | `emp.printSummary()` | *(без модификатора) / package-private* | **нет** | Аналогично строке C: метод без модификатора виден только внутри своего пакета. |
| H | `emp.validatePassword("secret")` | `private` | **нет** | Аналогично строке D: `private` методы можно вызвать только внутри класса `Employee`. |
