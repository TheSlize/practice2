# Задание 2.4 — Абстрактный класс vs Интерфейс

## Теоретическое напоминание

| Критерий | Абстрактный класс | Интерфейс |
|----------|-------------------|-----------|
| Отношение | «является» (is-a) | «умеет» (can-do) |
| Наследование | Одиночное (`extends`) | Множественное (`implements`) |
| Поля (состояние) | Могут быть любые | Только `static final` |
| Конструктор | Есть | Нет |
| Методы с реализацией | Да | `default`/`static` (с Java 8) |

**Правило выбора:**
- Если есть общие **поля** (состояние) и отношение «является» → **абстрактный класс**.
- Если нужна **множественная реализация** или отношение «умеет» → **интерфейс**.

## Ваша таблица

| Ситуация | Ваш выбор | Обоснование |
|----------|-----------|-------------|
| Все животные имеют имя и возраст, но звучат по-разному | **Абстрактный класс** | Есть общее состояние (поля: имя, возраст) и прослеживается чёткое отношение «является» (is-a животное). |
| Некоторые объекты (документ, изображение, настройки) можно сериализовать | **Интерфейс** | Объекты совершенно разной природы объединяются общим поведением/навыком. Это отношение «умеет» (can-do сериализоваться). |
| Все транспортные средства имеют скорость и топливо, но двигаются по-разному | **Абстрактный класс** | У них есть общие поля для хранения состояния (скорость, количество топлива) и отношение «является» (is-a транспортное средство). |
| Робот, посудомоечная машина и телефон — все умеют подключаться к Wi-Fi | **Интерфейс** | Классы из совершенно разных иерархий наследования. Их объединяет только способность («умеет» / can-do подключаться к сети). |
| Круг, квадрат и треугольник — все фигуры с площадью и периметром, у всех есть цвет | **Абстрактный класс** | Наличие цвета подразумевает поле (общее состояние), плюс все они подчиняются строгому отношению «является» (is-a фигура). |
| Класс должен одновременно уметь летать, плавать и ездить | **Интерфейсы** | В Java запрещено множественное наследование классов. Чтобы получить сразу несколько разных навыков («умеет» / can-do), нужно реализовать несколько интерфейсов. |
